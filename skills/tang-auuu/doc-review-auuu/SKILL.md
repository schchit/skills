# 飞书 AI 文档审查 Skill v2.8

## ⚠️ 重要约束（Agent 必须遵守）

1. **本 Skill 是强制执行规程，不是参考建议**
2. **必须按步骤执行，不得跳过任何步骤**
3. **必须输出结构化报告，不得自由发挥**
4. **必须尝试创建飞书文档，失败时报错**
5. **遇到问题必须按指定方式处理，不得自行决定**

**重要**：审查报告和汇总表是**必需的交付物**，不是可选功能。
- 在对话中输出报告 = 兜底输出（必须执行）
- 创建飞书报告文档 = 主要交付物（必须尝试，失败时明确报错）
- 更新汇总表 = 主要交付物（必须尝试，失败时明确报错）
- **如果因 API 未配置导致创建失败，必须明确报错并给出解决方案**
6. **工具不可用时必须引导用户，不得假设工具存在**

## 第0部分：能力边界（Agent 启动时检查）

### ✅ 支持的操作

| 操作 | 说明 | 依赖 |
|------|------|------|
| 审查单个文档 | 接收用户提供的文档内容或链接 | 无 |
| 审查多个文档 | 逐个处理多个文档 | 无 |
| 输出审查报告 | 在对话中返回 Markdown 报告 | 无 |
| 读取公开链接 | 尝试使用可用工具读取 | webReader/web_crawl |
| 创建飞书报告文档 | 使用飞书 API 创建报告 | 需配置飞书 API |
| 更新汇总表 | 更新飞书汇总表文档 | 需配置飞书 API |

### ❌ 不支持的操作

| 操作 | 原因 | Agent 应如何回复 |
|------|------|----------------|
| 扫描知识库文件夹 | API 无法按文件夹筛选 | "⚠️ 无法自动扫描文件夹。请提供具体文档链接或内容。" |
| 扫描云空间文件夹 | 需要 API 权限 | "⚠️ 无法自动扫描文件夹。请从飞书客户端复制文档链接。" |
| 读取需登录的文档 | 飞书文档需要登录访问 | "⚠️ 无法读取需要登录的文档。请设置文档为「链接可见」或提供文档内容。" |

## 第1部分：用户交互流程

### 步骤1：接收审查请求

```
1. 解析用户输入，支持以下格式：
   a) 文档链接：https://xxx.feishu.cn/docx/xxx
   b) 文档内容：用户直接粘贴的文本
   c) 多个链接/内容：批量提供

2. 验证/提取信息：
   - 文档链接或内容
   - 文档类型（文献调研/方案推演/实验设计/对外材料）

3. 如果用户未说明文档类型：
   - 询问："请问这是什么类型的文档？（文献调研/方案推演/实验设计/对外材料）"
   - 等待用户回复

4. 如果用户提供的是文件夹链接：
   - 回复限制说明
   - 引导用户提供具体文档链接或内容
```

---

### 步骤2：读取文档内容（多种方式）

```
优先级顺序：
1. 用户直接提供内容 → 直接使用
2. 用户提供链接 → 尝试读取
   a. 尝试 webReader 工具（如可用）
   b. 尝试 web_crawl 工具（如可用）
   c. 尝试 feishu_fetch_doc 工具（如可用）
3. 所有工具都失败 → 引导用户提供内容

读取失败处理：
如果工具返回 "content is empty" 或 "需要登录"：
回复："⚠️ 无法读取该文档。

可能原因：
1. 文档需要登录才能访问
2. 文档未设置为「链接可见」

解决方案：
1. 打开飞书文档，点击「分享」
2. 设置为「获得链接的任何人可查看」
3. 复制文档内容直接粘贴给我
4. 或提供其他公开链接

该文档已跳过，继续处理其他文档..."
```

### 步骤3：加载审查维度

```
根据文档类型，加载对应的 prompts 文件：

实验设计 → prompts/实验设计.md

如果 prompts 文件无法读取：
停止执行并通知用户："⚠️ 无法读取审查维度文件：{文件路径}，请确认文件存在"
```

### 步骤4：执行内容审查

```
对于每个文档：

1. 加载 AI 平台配置（必须执行）
   - 读取 config/ai_platforms.yaml 获取平台列表
   - 如果配置文件无法读取，使用默认平台列表

2. AI 对话链接检测（必须执行）
   - 检测规则见"AI对话链接检测"部分
   - 记录所有检测到的链接

3. 按维度逐项审查
   - 对每个维度给出：评级（✅/⚠️/❌）+ 具体发现
   - 发现问题按严重度分级：Critical / High / Medium / Low

4. 识别内容亮点
   - 列举1-3个做得好的地方
```

```
严格按照"审查报告模板"生成报告
```

**重要**：所有类型的审查记录都追加到同一个汇总表，按文档类型列区分。

## 第5部分：汇总表格式

### 汇总表模板

```markdown
| 审查日期 | 文档名称 | 文档类型 | 文档链接 | AI对话链接 | 审查状态 | 审查报告链接 |
|---------|---------|---------|---------|-----------|---------|-------------|
```

**说明**：汇总表文档只包含上述表格，无其他内容。首次创建时只有表头，后续追加记录行。

---

### 汇总表管理规则

#### 1. 汇总表位置

```
审查报告/
└── 审查汇总表.md  ← 唯一的汇总表，所有类型审查记录都在这里
```

**重要**：整个审查任务只有一个汇总表，不分类型/文件夹。

#### 2. AI对话链接列格式

- **无链接**：填写 `-`
- **单条链接**：直接填写完整 URL
- **多条链接**：使用 `<br>` 换行分隔，每行一个完整 URL
  ```
  https://claude.ai/chat/xxx<br>https://chatgpt.com/share/yyy
  ```

#### 3. 更新 vs 新建逻辑

**原则**：追加记录到唯一的汇总表，不新建

```
首次审查（任何类型）：
→ 创建汇总表：审查报告/审查汇总表.md

后续审查（所有类型）：
→ 追加记录到同一汇总表（打开文档 → 表格末尾追加行）
→ 不创建新汇总表文档
```

#### 4. 多文档/多类型审查的处理

| 场景 | 处理方式 |
|------|----------|
| 单次审查单个文档 | 追加一条记录 |
| 单次审查多个同类型文档 | 追加多条记录 |
| 单次审查多个不同类型文档 | 追加多条记录（文档类型列区分） |
| 跨多次审查 | 始终追加到同一汇总表 |

**示例**：
```
| 审查日期 | 文档名称 | 文档类型 | 文档链接 | AI对话链接 | 审查状态 | 审查报告链接 |
|---------|---------|---------|---------|-----------|---------|-------------|
| YYYY-MM-DD | {文档名} | {类型} | {链接} | {链接或-} | {✅/⚠/❌} | {报告链接} |

```

创建飞书报告文档和汇总表是**必需交付物**，使用 OpenClaw 内置的飞书扩展完成。

## 第8部分：Agent 执行检查清单

### 基础审查（必须完成）

```
开始审查前：
□ 已收到文档链接或内容
□ 确认文档类型

审查过程中：
□ 已读取文档内容（或使用用户提供的内容）
□ 已检测AI对话链接
□ 已加载对应审查维度文件
□ 已逐项评估每个维度
□ 已识别问题并分级
□ 已识别内容亮点

审查完成后：
□ 已生成结构化报告
□ 已在对话中输出报告

基础审查全部完成 → 报告"审查完成"
```

```
如果 feishu_doc 工具可用：

□ 已更新汇总表

如果 feishu_doc 工具不可用：

工具不可用不影响"审查完成"状态，但必须明确提示用户。
```
