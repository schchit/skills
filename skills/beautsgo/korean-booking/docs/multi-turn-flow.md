# 多轮对话流程设计

## 概述

booking-skill 现已支持标准的多轮对话流程，模仿 ChatGPT 的交互模式。用户可以通过自然语言指令逐步完成预约流程，无需点击按钮，所有操作通过对话驱动。

## 流程图

```
用户: "怎么预约 CNP 皮肤科？"
     ↓
[第1轮] Skill 返回：预约流程说明 + 操作建议
     ├─ 📖 预约流程详细信息
     └─ 💡 接下来你可以：
        • "打开链接"
        • "帮我预约"
        • "咨询客服"
     ↓
三个分支：

分支A - 打开链接：
用户选择: "打开链接"
     ↓
[第2轮] Skill 执行：
     ├─ 🌐 自动打开浏览器
     ├─ 📄 页面加载完成
     └─ 💡 接下来你可以：
        • "帮我预约"
        • "咨询客服"
        • "需要截图"

分支B - 直接预约：
用户选择: "帮我预约"
     ↓
[第3轮] Skill 执行：
     ├─ 🖱️ 自动点击预约按钮（Playwright）
     ├─ 📝 页面跳转到预约表单
     └─ 📋 返回：请填写表单信息

分支C - 咨询客服：
用户选择: "咨询客服"
     ↓
[第4轮] Skill 执行：
     ├─ 🌐 自动打开浏览器
     ├─ 🤖 自动点击咨询按钮（多选择器尝试）
     ├─ 💬 页面跳转到客服对话窗口
     └─ 📋 返回：客服对话指南
```

## 各轮详细说明

### 第1轮：查看预约流程（View）

**用户输入：**
```
"怎么预约 CNP 皮肤科？"
"CNP 皮肤科预约流程"
"如何预约 JD 皮肤科"
```

**Skill 行为：**
1. 解析用户输入，提取医院名称
2. 从 hospitals.json 匹配医院
3. 渲染预约流程模板（多渠道信息）
4. 返回友好的预约说明 + 建议下一步

**返回示例：**
```
🏥 CNP皮肤科狎鸥亭店 预约流程

在 BeautsGO 平台上，你可以通过以下渠道预约：

📱 App 预约：
1. 下载 BeautsGO App
2. 搜索"CNP皮肤科"
3. 点击"预约"按钮
4. 选择时间和服务

💻 网页端预约：
访问：https://i.beautsgo.com/cn/hospital/cnpskin-clinic
按照页面指引完成预约

... [更多渠道] ...

---
💡 接下来，你可以告诉我：
• "打开链接" - 我帮你打开医院的预约页面
• "帮我预约" - 我帮你自动点击预约按钮
• "咨询客服" - 我帮你联系客服咨询
• 或其他问题 😊
```

### 第2轮：打开浏览器（Open）

**用户输入：**
```
"打开链接"
"打开浏览器"
"打开医院页面"
```

**Skill 行为：**
1. 识别用户想打开浏览器
2. 使用 exec() 调用系统命令打开 hospital.url
   - macOS: `open "url"`
   - Windows: `start "url"`
   - Linux: `xdg-open "url"`
3. 等待浏览器打开
4. 返回确认信息 + 下一步建议

**返回示例：**
```
✅ 链接已打开！正在加载 CNP皮肤科狎鸥亭店 的预约页面...

页面地址：https://i.beautsgo.com/cn/hospital/cnpskin-clinic?from=google_map

你可以在浏览器中看到：
• 📍 医院地址和地图
• ⏰ 营业时间
• 💰 价格表和优惠
• 📷 医院环境照片
• 👨‍⚕️ 医生团队介绍
• ✅ 预约按钮

接下来，你可以：
• "帮我预约" - 我帮你自动点击预约按钮
• "需要截图" - 我可以帮你看看页面内容
• 或继续咨询其他问题 😊
```

### 第3轮：自动预约（Book）

**用户输入：**
```
"帮我预约"
"点击预约按钮"
"直接预约"
```

**Skill 行为：**
1. 识别用户想自动预约
2. 再次打开浏览器（确保页面可用）
3. 使用 Playwright 自动化：
   - 加载页面
   - 定位预约按钮：`.btns-right:has-text("预约面诊")`
   - 点击按钮
   - 等待页面跳转到预约表单
4. 关闭浏览器自动化（保留浏览器窗口供用户填表）
5. 返回表单填写说明

**返回示例：**
```
✅ 已帮你点击预约按钮！页面已跳转到预约表单。

CNP皮肤科狎鸥亭店 的预约页面已在浏览器中打开，请按照以下步骤继续：

📝 请填写以下信息：
• 您的姓名
• 联系电话
• 预约日期和时间
• 选择医生（如适用）
• 服务项目描述

💳 完成后点击"确认预约"或"提交"

如果需要帮助，随时告诉我！😊
```

### 第4轮（可选）：咨询客服（Consult）

**用户输入：**
```
"咨询客服"
"联系客服"
"我想咨询一下"
"在线客服"
```

**Skill 行为：**
1. 识别用户想咨询
2. 打开浏览器到医院页面
3. 使用 Playwright 自动化：
   - 尝试多个可能的咨询按钮选择器
   - 定位"咨询一下"、"在线客服"或其他客服按钮
   - 点击按钮
   - 等待页面跳转或弹窗出现
4. 返回咨询指南（成功或降级方案）

**返回示例（成功自动点击）：**
```
✅ 已帮你打开 CNP皮肤科狎鸥亭店 的咨询对话页面！

我已经：
• 打开了医院页面
• 自动点击了"咨询一下"按钮

现在页面应该已经跳转到在线客服对话窗口。你可以：
• 📝 在对话框中输入你的问题
• ❓ 询问价格、预约时间、医生等信息
• 💬 直接与客服沟通

还需要其他帮助吗？😊
```

**返回示例（降级方案）：**
```
⚠️ 自动点击咨询按钮失败，但页面已打开。

请手动点击以下位置的咨询按钮：
• 页面上方或右侧的蓝色"咨询一下"按钮
• 或右下角的"在线客服"按钮
• 点击后会打开客服对话窗口
```

**技术亮点：**
- 多选择器尝试：不依赖单一选择器，提高兼容性
- 可见性检查：确保元素在视口中可见再点击
- 备选方案：文本正则匹配 `/咨询|客服|在线|联系/i`
- 错误优雅降级：点击失败时返回手动操作指南

## 技术实现细节

### 意图识别（Intent Detection）

在 `api/skill.js` 中的 `detectIntent()` 函数：

```javascript
function detectIntent(query) {
  const q = query.toLowerCase()
  
  if (q.includes('打开')) return 'open'
  if (q.includes('点击') || q.includes('预约')) return 'book'
  if (q.includes('咨询') || q.includes('客服')) return 'consult'
  return 'view'  // 默认
}
```

### 浏览器自动化（Playwright）

在 `clickBookingButton()` 中使用 Playwright 自动化：

```javascript
async function clickBookingButton(url) {
  const browser = await playwright.chromium.launch({ headless: false })
  const page = await browser.newPage()
  await page.goto(url)
  
  // 关键选择器：BeautsGO 页面上的预约按钮
  const bookingButton = await page.locator('.btns-right:has-text("预约面诊")').first()
  await bookingButton.click()
  
  // 等待页面跳转到预约表单
  await page.waitForNavigation({ waitUntil: 'networkidle' })
  
  await browser.close()
}
```

### 上下文管理（Context）

虽然当前实现通过意图识别重新解析 query，但 Skill 已支持 `context` 参数以便日后扩展多轮状态管理：

```json
{
  "query": "帮我预约",
  "context": {
    "hospital": { "id": 2, "name": "CNP皮肤科狎鸥亭店", "url": "..." }
  }
}
```

## 用户体验优势

1. **无需点击 UI**：所有操作通过自然语言指令
2. **逐步引导**：每一步都有明确的下一步建议
3. **可控流程**：用户完全控制是否打开浏览器、自动化
4. **透明执行**：Skill 明确告诉用户发生了什么
5. **错误处理**：自动化失败时有优雅的降级方案

## 与标准 Skill 流程的区别

| 特性 | 标准 Skill | booking-skill |
|------|----------|--------------|
| 返回值 | 纯文本 | 文本 + 建议操作 |
| 交互 | 一次性返回 | 多轮对话 |
| 自动化 | 无 | 打开浏览器、点击按钮 |
| 用户控制 | 被动 | 主动（可选择是否自动化） |
| 需要点击 | 否 | 否（完全通过对话） |
