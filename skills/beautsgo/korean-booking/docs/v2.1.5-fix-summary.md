# 咨询按钮修复总结 - v2.1.5

## 🎯 问题
用户说"咨询客服"后，页面打开但**没有自动点击咨询按钮**，而是返回了手动操作指南。

## 🔍 根本原因分析

### 诊断步骤：
1. **初始测试** → Playwright 选择器无法找到 `.btns-consult` 元素
2. **HTML 快照分析** → 发现元素确实存在：
   ```html
   <uni-view class="btns-right btns-consult">
     <uni-text class="custom-icon custom-icon-kefu1"></uni-text>
     咨询一下
   </uni-view>
   ```
3. **等待时间测试** → 发现元素加载需要 **5+ 秒**（Vue 动态渲染）
4. **改进方案验证** → 增加等待时间到 8 秒，使用 `waitForSelector` ✅ 成功

## ✅ 解决方案（v2.1.5）

### 核心改进：
| 改进项 | v2.1.4 | v2.1.5 |
|------|--------|--------|
| 等待时间 | 5 秒 | 8 秒 |
| 选择器策略 | 直接尝试 6 个选择器 | `waitForSelector` + 混合策略 |
| 降级方案 | DOM evaluate | 精确文本匹配 + Playwright 选择器 |
| 日志清晰度 | 基础 | 详细区分不同失败场景 |

### 代码改动：
```javascript
// 关键改进 1: 增加等待和 waitForSelector
await page.waitForTimeout(8000)
await page.waitForSelector('.btns-consult', { timeout: 10000 })

// 关键改进 2: 优先使用 class 选择器
const button = document.querySelector('.btns-consult')

// 关键改进 3: 精确文本匹配作为备选
if (text === '咨询一下' && el.offsetParent !== null)
```

## 📝 技术细节

### BeautsGO 页面特性：
- 框架：Vue.js + uni-app（微信小程序框架）
- 组件：`<uni-view>` 自定义组件（非标准 HTML）
- 加载：动态渲染，需要 5+ 秒完全初始化
- 咨询按钮：`class="btns-right btns-consult"`

### 为什么之前失败：
- 原有等待时间（5秒）可能不足
- Playwright 的文本选择器 `text=/咨询一下/` 对动态组件不可靠
- 缺少 `waitForSelector` 主动等待元素

### 为什么现在成功：
- 8 秒等待确保 Vue 完全初始化
- `waitForSelector` 主动检测元素何时出现
- 优先使用 class 选择器 `.btns-consult`（比文本匹配更可靠）
- 完善的降级策略
