# MockLab - 智能 Mock 数据生成器

**不用等网关，自己搭一个。**

---

## 是什么

对接第三方接口时，测试环境迟迟不就绪？

MockLab 是一个 AI 驱动的本地 Mock 接口服务。上传任意格式的接口文档（Markdown、Word、Java 源码），AI 自动理解文档、生成 Mock Schema、启动本地服务器，**5 分钟内**可以开始调测试接口。

---

## 解决的问题

| 原来怎么做 | 用 MockLab 怎么做 |
|-----------|-----------------|
| 等第三方网关就绪（3天～数周） | 拿到文档，5分钟上线 |
| 手动配 Mock（1～2天/接口） | AI 自动解析，无需手动 |
| 手动维护跨接口状态（易出错） | `is_ref` + `state:` 机制自动处理 |
| 每次文档更新全部重来 | AI 自动重新解析，即时更新 |
| 边界测试（延迟/错误/超时）无法测 | 内置延迟模拟 + 错误注入 |
| 前端想模拟真实接口行为 | 请求镜像 + 动态响应 |

---

## v1.1 新功能（2026-04-02）

- **延迟模拟** — 模拟真实网络延迟，支持固定值和随机范围
- **错误注入** — 支持自定义错误码、错误消息、触发概率
- **请求镜像** — 把请求参数原样返回到响应，接口更逼真
- **动态响应** — 根据请求内容（body/path/query/header/state）动态生成响应
- **条件字段** — 支持 `when` 条件，满足条件才返回字段
- **UI 配置** — 延迟和错误注入可在网页上直接配置，无需改 Schema

## 功能特性

- **AI 智能解析** — 支持 Markdown、Word、Java 源码、纯文本、URL，无需手动配置
- **真实数据生成** — 合法身份证、真实号段手机号、合规车牌号、金额、利率等
- **嵌套结构支持** — reqData、vehicle、nodeList 等 2～3 层嵌套自动处理
- **加密字段处理** — reqData / ciphertext 自动解析，body: 可下钻取嵌套值
- **跨接口状态关联** — `is_ref: true` 存入状态，`state:` 取值，无需手动传参
- **状态持久化** — 重启不丢失，无需重新配置
- **多项目支持** — 一个服务承载多个第三方接口，随时切换
- **实时 UI** — 请求编辑 + 发送 + 格式化响应展示 + 延迟/错误配置
- **热更新** — 修改配置无需重启服务

## 字段规则一览

### 基础规则

| 字段特征 | 规则写法 | 生成结果 |
|---------|---------|---------|
| 手机号 | `phone` | 11位真实号段（13x/15x/18x/199） |
| 身份证号 | `idcard` | 18位合法身份证（含校验位） |
| 姓名 | `name` | 中文姓名 |
| 车牌号 | `plate` | 合规车牌 |
| 金额 | `amount:lo-hi` | 指定范围随机金额 |
| 利率 | `rate:lo-hi` | 指定范围年利率 |
| 随机数 | `random:6` | 6位随机数字 |
| 时间戳 | `timestamp:ms` | 毫秒级时间戳 |
| Token | `token:32` | 32位随机字符串 |
| 固定值 | `fixed:VALUE` | 固定返回值 |
| 枚举值 | `enum:a,b,c` | 从枚举列表随机 |
| 自增序列 | `sequence:key` | 日期+6位自增序号 |
| 跳过 | `skip` | 不生成（sign 等） |
| 嵌套对象 | `nested:schema名` | 引用 inner_schemas |
| 对象数组 | `array:schema名` | 生成数组 |

### 动态响应规则（v1.1 新增）

| 规则写法 | 说明 | 示例 |
|---------|------|------|
| `mirror` | 取请求 body 中同名字段 | 请求 `{"order_id": "A123"}` → 响应 `"order_id": "A123"` |
| `body:data.order_id` | 取请求 body 嵌套字段 | 取 `body.data.order_id` 的值 |
| `path:order_id` | 取 URL 路径参数 | `/order/{order_id}` 中实际值 |
| `query:page` | 取 URL query 参数 | `?page=2` 中 page 值 |
| `header:Authorization` | 取请求头 | 取 Authorization 头 |
| `state:token` | 取跨次调用状态 | 取上次接口存入的 token |
| `when` 条件 | 满足条件才返回 | `{"field": "body.is_vip", "op": "eq", "value": true}` |

### when 支持的操作符

`eq` / `ne` / `gt` / `lt` / `contains` / `exists` / `not_exists`

## 接口级配置

在 Schema JSON 中，每个接口顶层可以设置：

```json
{
  "name": "提交订单",
  "path": "/order/create",
  "method": "POST",
  "delay": "200-500",
  "error": "500@0.05:ERR_SYS:系统异常",
  "req_fields": [...],
  "resp_fields": [...]
}
```

### 延迟配置（delay）

| 写法 | 效果 |
|------|------|
| `"delay": "300"` | 固定 300ms |
| `"delay": "200-500"` | 随机 200~500ms |
| `"delay": "1000-3000"` | 大压力测试 1~3s |

### 错误注入配置（error）

格式：`错误类型[@概率[:错误码[:消息]]]`

| 示例 | 说明 |
|------|------|
| `"error": "500"` | 100% 返回 HTTP 500 |
| `"error": "timeout"` | 100% 超时（等待 30s）|
| `"error": "network_error"` | 100% 返回 HTTP 502 |
| `"error": "rate_limit"` | 100% 返回 HTTP 429 |
| `"error": "500@0.05"` | 5% 概率返回 HTTP 500 |
| `"error": "500@1.0:ERR001:自定义错误"` | 100% HTTP 500，自定义 code 和消息 |
| `"error": "timeout@0.5::自定义超时消息"` | 50% 超时，自定义消息 |

**推荐在 UI 上直接配置**（http://localhost:18080），支持多字段输入，无需记忆格式。

## 状态关联（is_ref / state:）

### 存入状态

响应字段加 `is_ref: true`，生成后自动存入状态表：

```json
{
  "name": "accessToken",
  "rule": "token:32",
  "is_ref": true
}
```

### 从状态取值

请求字段用 `state:字段名`，自动取上次存入的值：

```json
{
  "name": "accessToken",
  "rule": "state:accessToken"
}
```

适用于：Token、会话 ID、订单号等需要在多次调用间传递的值。

## 常见问题

**Q: 签名字段怎么处理？**
A: sign 等鉴权字段自动 skip，不会出现在请求体中。

**Q: 加密字段（reqData、data、ciphertext）怎么处理？**
A: 模型从文档提取明文结构写入 inner_schemas。请求时 reqData 字符串会自动解析为 JSON，body: 可下钻取其中字段值。

**Q: 状态关联怎么工作？**
A: 响应字段标记 `is_ref: true` 后自动存入状态表，后续接口用 `state:字段名` 引用时自动填充。

**Q: 支持多少个项目？**
A: 没有上限，UI 右上角随时切换。

**Q: 延迟和错误注入怎么配置？**
A: 两种方式：① 在 UI 上直接填写（推荐）；② 在 Schema JSON 中写入 delay / error 字段。
