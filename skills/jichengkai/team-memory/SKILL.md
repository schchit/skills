# Team Memory - 团队记忆系统

## 概述

为技术管理者设计的双向记忆系统：
- **记录成员**：观察行为、成长轨迹
- **记录自己**：管理动作、沟通反思

## 核心功能

1. **成员时间轴**：按时间顺序记录每个人的成长轨迹
2. **三种记录类型**：观察、对话、管理动作
3. **自动追踪**：承诺 → 提醒 → 反馈 → 反思
4. **场景化输出**：年终总结、1:1准备、晋升评估

## 数据结构

```
~/.config/opencode/skills/team-memory/
├── SKILL.md
├── skill-config.yaml
└── data/
    ├── members/                    # 成员时间轴
    │   ├── 张三-时间轴.md
    │   └── 模板-成员时间轴.md
    ├── manager-journal/            # 管理反思
    │   ├── tracker/               # 自动追踪
    │   └── 月度复盘/
    └── insights/                  # 洞察分析
```

## 三种记录类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **观察** | 成员做了什么 | `记录 张三 今天 技术突破 优化接口` |
| **对话** | 我对成员说了什么 | `记录 /d/ 张三 今天 1on1 鼓励改进` |
| **管理** | 我的管理决定 | `记录 /m/ 张三 今天 晋升` |

## 快速开始

### 1. 初始化
```bash
# Windows PowerShell
mkdir -p "$env:USERPROFILE\.config\opencode\skills\team-memory\data\members"

# macOS/Linux
mkdir -p ~/.config/opencode/skills/team-memory/data/members
```

### 2. 配置成员
编辑 `skill-config.yaml`，添加团队成员映射。

### 3. 记录
```
记录 张三 今天 技术突破 优化接口性能提升10倍
记录 /d/ 张三 今天 1on1 鼓励沟通改进，承诺分配云原生任务
```

## 记录格式

### 观察记录
```markdown
### {日期}
#### {时间} - {标题}
**事件**: {发生了什么}
**类别**: 技术能力 / 协作沟通 / 项目交付 / 团队影响 / 成长潜力
**评价**: ⭐⭐⭐⭐ 良好
**标签**: #{标签}

**观察笔记**: {分析}
**后续行动**: - [ ] {待办}
```

### 对话记录
```markdown
### {日期}
#### {时间} - 1:1沟通
**事件**: 和{姓名}1:1
**标签**: #对话 #1on1

**我说了**: {要点}
**我的承诺**: - [ ] {待办}

**事后反思**:
- ✅ 有效：{效果}
- ❌ 需改进：{改进点}
```

## 追踪机制

### 追踪循环
```
承诺 → 到期提醒 → 反馈效果 → 更新反思 → 下次改进
```

### 追踪时机
| 类型 | 时机 | 问题 |
|------|------|------|
| 承诺 | 到期前3天、当天、到期后3天 | 进展如何？ |
| 对话效果 | 7天后 | 有什么变化？ |
| 管理动作 | 30天后 | 成员状态？ |

## 使用节奏

- **每日**: 记录1-3条（2-3分钟）
- **每次对话后**: 追加对话记录（1分钟）
- **每周五**: 查看本周记录
- **每月**: 管理复盘（15分钟）

## 隐私安全

1. 本地存储，不上传云端
2. 使用 member-XXX 编号代替真名
3. 仅记录工作相关表现
4. 离职成员归档到 archive/
