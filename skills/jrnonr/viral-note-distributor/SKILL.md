# 爆款内容多平台分发 Skill

## 核心定位

用户输入（可能不连贯/不结构化）→ Query重写整理 → 识别意图 → 只生成需要发布的平台内容

---

## Query 重写

当用户输入不连贯、不结构化时，先重写整理，再进行后续流程。

### 需要重写的情况

| 用户输入（原始） | 重写后 |
|-----------------|--------|
| "就是那个openclaw啊 用起来还不错 但是配置挺麻烦的 国内用有点卡 怎么说呢 就是感觉还行吧" | 话题：OpenClaw使用体验；感受：配置复杂但用起来方便，国内略卡，token消耗需预算；核心观点：值得折腾但要有预算 |
| "openclaw体验" | 话题：OpenClaw使用体验；需提取更多信息点 |

### 重写输出格式

```json
{
  "topic": "明确的话题",
  "key_points": ["核心观点1", "核心观点2", "核心观点3"],
  "angle": "用户视角/立场",
  "target_audience": "给谁看的",
  "mood": "整体基调（评测/分享/吐槽/推荐等）",
  "original_raw": "原始输入（保留参考）"
}
```

### 重写判断逻辑

```
用户输入
     ↓
判断是否需要重写？
├─ 字数 < 20字？ → 需要补充提问
├─ 内容不连贯/碎片化？ → 重写整理
├─ 缺话题/观点/受众？ → 追问补充
└─ 已经结构化清晰？ → 直接下一步
     ↓
重写后继续流程
```

如果重写后信息仍不足，主Agent应主动追问用户补充关键信息（话题、观点、受众等）。

## 意图识别与平台匹配

### 用户可能说的话

| 用户说 | 意图 | 只生成 |
|--------|------|--------|
| "帮我写小红书帖子" | 单平台 | 小红书 |
| "生成抖音文案" | 单平台 | 抖音 |
| "发到小红书和B站" | 指定多平台 | 小红书 + B站 |
| "帮我分发到4个平台" | 全平台 | 全部4个 |
| "发到全平台" | 全平台 | 全部4个 |
| "帮我改写这篇笔记" | 默认全平台 | 全部4个 |

### 平台关键词

**单平台关键词（触发单平台）：**
- 小红书/小红书帖子/小红书笔记/xhs
- 抖音/抖音脚本/抖音文案/短视频
- B站/b站/bilibili/专栏
- 公众号/微信公众号/微信文章

**多平台关键词（触发全平台）：**
- 多平台/全平台/4个平台
- 发到小红书、抖音、B站和公众号

### 判断逻辑

```
用户输入原始内容
     ↓
扫描平台关键词
     ↓
┌─────────────────────────────────┐
│ 找到"小红书"相关词？ → 加入列表  │
│ 找到"抖音"相关词？ → 加入列表   │
│ 找到"B站"相关词？ → 加入列表    │
│ 找到"公众号"相关词？ → 加入列表 │
│ 找到"多平台/全平台"关键词？     │
│   → 列表设为全部4个平台        │
│ 什么都没找到？ → 默认全平台     │
└─────────────────────────────────┘
     ↓
只对列表中的平台生成内容
```

## 工作流

```
用户输入（原始、不连贯）
     ↓
【Query重写】整理成结构化内容
     ↓
【意图识别】分析用户说哪些平台、要不要封面图
     ↓
【主Agent】分析内容核心 + 提取关键信息点
     ↓
┌─────────────────────────────────────┐
│  小红书适配版（如果需要）            │
│  抖音适配版（如果需要）              │
│  B站适配版（如果需要）              │
│  公众号适配版（如果需要）            │
└─────────────────────────────────────┘
     ↓
【可选】--with-cover → 封面图生成提示词
     ↓
【主Agent】汇总输出（只输出需要的平台）
```

## 子 Agent 调用方式

### 调用流程

主 Agent 向需要的子 Agent 发送相同的上下文信息：

```json
{
  "task": "platform_adaptation",
  "platform": "具体平台名",
  "original_content": "用户输入的原始笔记全文",
  "content_summary": "主Agent提炼的内容核心要点",
  "key_points": ["关键信息点1", "关键信息点2"],
  "target_audience": "目标受众描述",
  "with_cover": true/false,
  "platform_style_guide": "该平台的风栺指南",
  "hooks_library": "爆款钩子库"
}
```

### 子 Agent 输出格式

```json
{
  "platform": "平台名",
  "title": "平台适配标题",
  "cover_suggestion": "封面文案建议（10字内）",
  "cover_image_prompt": "封面图生成提示词（--with-cover时）",
  "body": "正文内容",
  "hashtags": ["标签1", "标签2", "标签3"],
  "publish_tips": "发布建议（时间、注意事项）"
}
```

### 并行执行

需要的子 Agent 并行派生，全部完成后主 Agent 汇总：

```
意图识别 → 确定需要哪些平台
     ↓
sessions_spawn(需要的平台1) ─┐
sessions_spawn(需要的平台2) ─┤  并行
sessions_spawn(需要的平台3) ─┤
sessions_spawn(需要的平台4) ─┘
     ↓
主Agent汇总 → 只输出需要的平台
```

## 使用方式

### 单平台分发

```
用户：帮我写一篇小红书帖子，内容是：[笔记内容]
用户：生成抖音文案：[内容]
用户：发到B站：[内容]
用户：公众号：[内容]
```

### 多平台分发

```
用户：帮我把这篇笔记分发到小红书、抖音、B站和公众号
用户：发到全平台：[内容]
```

### 带封面图

```
用户：帮我写小红书帖子 --with-cover
用户：发到全平台 --with-cover
```

## 平台输出规格

详见 `references/platform-styles.md`

| 平台 | 标题字数 | 正文字数 | 标签数 |
|------|---------|---------|--------|
| 小红书 | ≤20字 | 800-1000字 | 5-8个 |
| 抖音 | ≤15字 | 300-500字 | 3-5个 |
| B站 | ≤25字 | 1000-1500字 | 5-8个 |
| 公众号 | 20-30字 | 1500-2000字 | 1-2个 |
