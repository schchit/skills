# super_memori — Release Status Model

## 3.0.x meaning
`3.0.x` means: working v3 baseline.
That includes:
- public four-command surface
- lexical index
- health check
- queue/freshness
- honest degraded semantic reporting

It does **not** require full semantic activation.

## 3.1.x meaning
`3.1.x` means: full-hybrid-ready skill spec.
That includes:
- stronger design references for semantic completion
- explicit acceptance criteria for hybrid mode
- deterministic implementation order for weaker models

It still does **not** imply the current host has semantic dependencies installed unless verified.
It is a documentation/spec maturity state, not a runtime activation state.

## 4.0.0 meaning
`4.0.0` should be reserved for a release where the runtime itself can truthfully claim:
- hybrid mode implemented
- semantic indexing active
- hybrid retrieval verified
- health/retrieval/docs all aligned

For the short operational path to that state, see `roadmap-to-4.0.0.md`.
