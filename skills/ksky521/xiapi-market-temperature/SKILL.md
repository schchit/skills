# 市场温度分析 Skill

分析A股市场温度指标，判断市场现状和变化趋势，识别买卖信号。

## Overview（功能概述）

使用大虾皮 API 获取市场温度四大指标（估值温度、恐贪指数、趋势温度、动量温度），通过多维度分析判断市场冷热程度和投资时机。

## When to Use（何时使用）

当用户需要以下功能时触发：

- 判断市场整体冷热程度
- 分析市场情绪和估值水平
- 识别潜在的买卖信号
- 了解市场中期趋势强度

**具体触发词**：市场温度、市场估值、恐贪指数、市场情绪、趋势温度、动量温度、估值分析、情绪分析

## When Not to Use（何时不使用）

以下场景不应使用本 Skill：

- 个股层面的买卖点、基本面或财报解读
- 行业板块层面的分析
- 债券、基金、期货、外汇等非市场温度分析
- 需要实时盘中信号或自动交易指令的请求

## Process（流程主体）

### Step 0: 前期准备

**触发条件**：首次使用或 Token 未配置

**跳过条件**：Token 已配置且有效，确认已经配置了Token则跳过该步骤

**完成标准**：API 调用正常，无认证错误

**执行步骤**：

**步骤 0.1：检查 Token 配置状态**

```bash
npx daxiapi-cli@latest config get token
```

**步骤 0.2：如未配置，获取 Token**

1. 提示用户访问 [daxiapi.com](https://daxiapi.com) 个人主页
2. 开通 API Token 功能
3. 获取生成的 Token

**步骤 0.3：配置 Token**

```bash
# 方式一：通过 CLI 配置（推荐）
npx daxiapi-cli@latest config set token YOUR_TOKEN_FROM_DAXIAPI

# 方式二：设置环境变量
export DAXIAPI_TOKEN=YOUR_TOKEN_FROM_DAXIAPI
```

**步骤 0.4：验证配置**

```bash
npx daxiapi-cli@latest market temp
```

如返回正常市场温度数据，则配置成功。

### Step 1: Agent 循环（获取+分析）

**目标**：从多个维度获取市场温度数据并进行深入分析

**推荐维度**（根据用户需求选择 3-4 个）：

| 维度         | CLI 命令                              | 分析重点               | 优先级 |
| ------------ | ------------------------------------- | ---------------------- | ------ |
| 估值温度     | `npx daxiapi-cli@latest market temp`  | 长期估值水平           | 高     |
| 恐贪指数     | `npx daxiapi-cli@latest market temp`  | 短期情绪极端值         | 高     |
| 趋势温度     | `npx daxiapi-cli@latest market temp`  | 中期趋势强度           | 高     |
| 动量温度     | `npx daxiapi-cli@latest market temp`  | 市场整体动能           | 中     |

**核心指标说明**：

**四大温度指标**：

| 指标       | 定义                         | 时间维度 | 关键阈值                          |
| ---------- | ---------------------------- | -------- | --------------------------------- |
| 估值温度   | 市盈率历史百分位             | 长期     | <20 低估，20-70 中性，>70 高估    |
| 恐贪指数   | 市场贪婪恐惧程度             | 短期     | 0-10 极度恐惧，90-100 极度贪婪    |
| 趋势温度   | 股价高于 MA60 占比           | 中期     | <20 低迷，40-60 中性，>80 过热    |
| 动量温度   | 动量值中位数百分位           | 中短期   | <20 偏弱，40-60 正常，>80 强势    |

**循环流程**：

**每次循环包含**：

1. **获取数据**
    - 执行 CLI 命令：`npx daxiapi-cli@latest market temp`
    - 检查数据完整性（日期、四大温度指标）
    - 处理异常情况（非交易日、数据延迟等）

2. **分析数据**
    - 输出最新数据表格
    - 分析最近 20 日指标变动趋势
    - 识别买入信号（恐贪、趋势、动量）
    - 记录重要发现
    - 特别注意：**出现数字为0的情况，可能是数据没有更新，比如恐贪数据是在交易日晚8点之后更新**

**买入信号判断规则**：

| 指标         | 买入信号条件 | 说明                         |
| ------------ | ------------ | ---------------------------- |
| 恐贪指数     | < 30         | 市场情绪恐惧，可能接近底部   |
| 趋势温度     | < 40         | 市场处于弱势区，可能接近底部 |
| 动量温度     | < 40         | 动量偏弱，超跌反弹机会       |

**灵活性**：

- ✅ 可根据数据质量调整分析深度
- ✅ 可根据初步结果调整重点指标
- ✅ 可根据用户需求调整建议重点

**完成标准**：

- 至少完成 3 个温度指标的分析
- 买入信号已识别和记录
- 数据异常已处理
- 关键发现已记录

## Common Pitfalls（常见陷阱）

### 数据陷阱

**陷阱 1：数据时间不一致**

- 问题：使用不同时间点的温度数据进行对比
- 后果：结论错误，误导投资决策
- 避免方法：确保所有数据时间点一致，标注数据时间

**陷阱 2：数据更新延迟**

- 问题：在非交易日或盘前查询，数据可能过旧
- 后果：分析不反映最新市场状态
- 避免方法：提醒用户数据每日收盘后更新

### 分析陷阱

**陷阱 3：单一指标决策**

- 问题：仅根据一个温度指标做决策
- 后果：判断片面，可能失误
- 避免方法：综合多个指标判断，避免单一指标误导

**陷阱 4：短期长期混淆**

- 问题：混淆估值温度（长期）和恐贪指数（短期）的作用
- 后果：策略错误，操作失误
- 避免方法：明确区分各指标的时间维度和用途

**陷阱 5：过度拟合历史**

- 问题：认为历史百分位一定会均值回归
- 后果：低估趋势延续的可能性
- 避免方法：承认市场可能出现极端情况

### 表达陷阱

**陷阱 6：绝对化表述**

- 问题：使用"一定"、"必然"、"肯定"等词
- 后果：误导用户，违反合规要求
- 避免方法：使用"可能"、"大概率"、"或许"等词

**陷阱 7：预测性表述**

- 问题：对市场走势做出确定性预测
- 后果：误导用户，违反合规要求
- 避免方法：改为情景分析或概率判断

## Gotchas（避坑案例）

- **401 认证失败（Token 缺失或失效）**
  - 错误表现：`401 Unauthorized` 或 `Authentication failed`
  - 修复动作：执行 `npx daxiapi-cli@latest config get token` 检查配置；若为空或失效，执行 `npx daxiapi-cli@latest config set token YOUR_TOKEN_FROM_DAXIAPI` 重新配置

- **空数据返回（非交易日/更新延迟）**
  - 错误表现：返回空数据或数据日期过旧
  - 修复动作：向用户说明数据时效限制，建议在收盘后再查询

- **429 请求频率超限**
  - 错误表现：`429 Too Many Requests`
  - 修复动作：等待 30-60 秒后重试

## Key Principles（重要原则）

### 分析原则

1. **结论先行**：开头即给出市场状态判断
2. **数据驱动**：所有结论基于具体温度值
3. **多维分析**：综合多个指标判断，避免单一视角
4. **趋势结合**：不仅看当前值，还要看变化趋势

### 合规原则

5. **风险提示优先**：每份报告必须包含风险提示
6. **免责声明**：所有报告必须包含免责声明
7. **中性表述**：不使用绝对化词汇

### 表达原则

8. **诚实透明**：承认不确定性和数据局限性
9. **用户友好**：使用通俗易懂的语言解释专业术语
