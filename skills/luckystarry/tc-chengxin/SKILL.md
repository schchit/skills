## ⚠️ 核心约束(违反将导致虚假信息)

**本技能的所有数据必须来自脚本执行结果,禁止大模型:**

| ❌ 禁止行为 | ✅ 正确做法 |
|------------|------------|
| 自行编造车次、价格、余票、酒店信息 | 调用脚本获取真实数据 |
| 修改、简化或重新格式化脚本输出 | **原样输出脚本的全部输出** |
| 脱离脚本数据凭空创造预订链接 | 直接使用 API 返回的链接 |
| 过滤或省略用户输入的日期信息 | 保留完整 query 传递给脚本 |
| ❌ **省略或替换预订链接** | ✅ **必须完整渲染 API 返回的所有预订链接** |

**原因:** 旅游数据实时变化,只有脚本调用同程后端 API 才能获得准确信息和有效链接。

> 🚨 **预订链接强制要求**:
> - **必须完整输出**脚本返回的所有预订链接(PC 端 + 手机端)
> - **禁止省略**任何资源的链接(车次、航班、酒店、景区等)
> - **禁止替换**为其他链接或文本
> - **禁止伪造**不存在的链接
> - **不允许忽略预订链接**--每个车次/航班/酒店/景点都必须附带完整的预订链接
> - 链接是用户完成预订的唯一有效路径,省略将导致用户无法购票

> ⚠️ **严重警告**:忽略或伪造预订链接将导致用户无法完成预订,属于严重错误行为!

## 🚗 交通资源智能查询 (traffic-query.js)

> 🚨 **智能推荐**:当用户**未明确指定交通方式**时使用此接口!

**智能交通查询能力**,同时返回机票、火车票、汽车票等多种交通方式,方便用户对比选择。

### 使用场景

| 场景 | 示例 | 推荐接口 |
|------|------|---------|
| 用户明确说"火车票" | "北京到上海的火车票" | `train-query.js` ✅ |
| 用户明确说"机票" | "明天飞北京的航班" | `flight-query.js` ✅ |
| 用户明确说"汽车票" | "苏州到南京的汽车" | `bus-query.js` ✅ |
| **用户未指定交通方式** | "北京到上海怎么走" | **`traffic-query.js`** ✅ |

### 合法参数组合

| 组合 | 参数示例 | 说明 |
|------|---------|------|
| 出发地 + 目的地 | `--departure "北京" --destination "上海"` | 按城市查询 |

### 参数说明

| 参数 | 说明 |
|------|------|
| `--departure <城市>` | 出发地城市(必填) |
| `--destination <城市>` | 目的地城市(必填) |
| `--extra <补充信息>` | 额外信息(日期、偏好等) |

### extra 参数示例

- `"明天"` - 明天的班次
- `"自驾"` - 偏好自驾路线
- `"明天 高铁优先"` - 多条件组合

### 调用示例

```bash
# 北京到上海,智能推荐交通方式
node scripts/traffic-query.js --departure "北京" --destination "上海" --channel webchat --surface webchat

# 明天出发
node scripts/traffic-query.js --departure "北京" --destination "上海" --extra "明天" --channel webchat --surface webchat

# 包含自驾偏好
node scripts/traffic-query.js --departure "苏州" --destination "南京" --extra "自驾" --channel webchat --surface webchat
```

### 输出结果

**同时展示多种交通方式**:
- 🚄 火车票(最多 5 趟)
- ✈️ 机票(最多 5 班)
- 🚌 汽车票(最多 5 班)

**每种交通方式都包含**:
- 完整班次/航班信息
- PC 端和手机端预订链接
- 链接格式:`[PC 端](url)` 和 `[手机端](url)`(微信渠道为纯文本 URL)

### 调用优先级

> ⚠️ **重要**: `traffic-query.js` 的调用优先级**低于**专用查询接口!

**正确用法:**
1. 用户说"火车票" → 使用 `train-query.js`
2. 用户说"机票" → 使用 `flight-query.js`
3. 用户说"汽车票" → 使用 `bus-query.js`
4. 用户**未指定**交通方式 → 使用 `traffic-query.js`

## ✈️ 机票专用查询 (flight-query.js)

> 🚨 **优先级最高**:查询机票时,**必须优先使用此方法**!

**更精准的机票查询能力**,支持航班号精确查询、特价机票查询、多列表推荐。

### 合法参数组合

| 组合 | 参数示例 | 说明 |
|------|---------|------|
| 出发地 + 目的地 | `--departure "北京" --destination "上海"` | 按城市查询 |
| 航班号 | `--flight-number "CA1234"` | 精确查航班 |
| 出发地 + lowPrice | `--departure "北京" --low-price` | 查询特价机票 |

### 参数说明

| 参数 | 说明 |
|------|------|
| `--departure <城市>` | 出发地城市 |
| `--destination <城市>` | 目的地城市 |
| `--flight-number <航班号>` | 航班号 (如 CA1234) |
| `--extra <补充信息>` | 额外信息 (日期、偏好等) |
| `--low-price` | 查询特价机票 (仅出发地时有效) |

### lowPrice 参数说明

当用户**只提供出发地,没有指明目的地**时,可以使用 `--low-price` 参数查询从该地出发到全国各地的特价机票。

**触发关键词:**
- "特价机票"、"低价机票"、"便宜机票"
- "从 XX 出发哪里便宜"、"XX 出发的优惠机票"
- "清明节/五一/春节 特价机票"(节假日低价推荐)

**示例场景:**
| 用户查询 | 调用方式 | 说明 |
|---------|---------|------|
| "北京出发的特价机票" | `--departure "北京" --low-price` | 查询北京出发低价航线 |
| "清明节北京出发的特价机票" | `--departure "北京" --low-price --extra "清明节"` | 带节假日条件 |
| "从上海飞哪里便宜" | `--departure "上海" --low-price` | 查询低价目的地推荐 |
| "周末从广州出发哪里好玩" | `--departure "广州" --low-price --extra "周末"` | 带时间偏好 |

**引导话术:**
```
💡 您只提供出发地,没有指定目的地。
您可以:
1. 补充目的地 (如:北京到上海)
2. 查询特价机票 (查看从该地出发到全国各地的优惠)
```

### 调用示例

```bash
# 北京到上海，明天的机票
node scripts/flight-query.js --departure "北京" --destination "上海" --extra "明天" --channel webchat --surface webchat

# 查询特定航班
node scripts/flight-query.js --flight-number "CA1234" --channel webchat --surface webchat

# 查询从北京出发的特价机票
node scripts/flight-query.js --departure "北京" --low-price --channel webchat --surface webchat

# 查询清明节北京出发的特价机票
node scripts/flight-query.js --departure "北京" --low-price --extra "清明节" --channel webchat --surface webchat
```

### 参数不完整时的处理

如果用户只提供了出发地,没有目的地:**提示用户补充目的地或使用 lowPrice 查询特价机票**。

### 输出结果要求

**必须完整输出所有航班的预订链接**:
- 每个航班都有 PC 端和手机端预订链接
- 链接格式:`[PC 端](url)` 和 `[手机端](url)`(微信渠道为纯文本 URL)
- **禁止省略任何航班的链接**(省略 = 用户无法购票)
- **禁止伪造不存在的链接**

| `--departure <城市>` | 出发地城市 |
| `--destination <城市>` | 目的地城市 |
| `--departure-station <站>` | 出发站(精确) |
| `--arrival-station <站>` | 到达站(精确) |
| `--train-number <车次>` | 车次号(如 G1234) |
| `--extra <补充信息>` | 额外信息(日期、偏好等) |

### extra 参数示例

- `"明天"` - 明天的车次
- `"高铁"` - 只查高铁
- `"动车"` - 只查动车
- `"一等座"` - 优先一等座
- `"明天 高铁 赏花专线"` - 多条件组合

### 参数不完整时的处理

如果用户只提供了部分参数,**必须提醒用户补齐**:

```
⚠️ 参数不完整,请提供以下组合之一:
  1. 出发地 + 目的地
  2. 车次号
  3. 出发站 + 到达站
```

### 输出结果要求

**必须完整输出所有资源的预订链接**:
- 每个资源(车次、航班、酒店、景区等)都有 PC 端和手机端预订链接
- 链接格式:`[PC 端](url)` 和 `[手机端](url)`(微信渠道为纯文本 URL)
- **禁止省略任何资源的链接**(省略 = 用户无法购票)
- **禁止伪造不存在的链接**

## 🧳 旅行资源专用查询 (travel-query.js)

> 🚨 **度假产品推荐**:当用户有明确旅游意向时作为**补充推荐**使用!

**专业的度假产品查询能力**,支持自由行、跟团游等多种旅游产品推荐。

### 使用场景

| 场景 | 示例 | 推荐接口 |
|------|------|---------|
| 用户查询"三亚自由行" | "三亚自由行套餐" | **`travel-query.js`** ✅ |
| 用户查询"云南旅游团" | "云南 6 天 5 晚跟团游" | **`travel-query.js`** ✅ |
| 用户有明确意向 | "我想去三亚玩" | **`travel-query.js`** + 单品资源 ✅ |
| 用户查询机票/酒店 | "三亚的酒店" | `hotel-query.js` + `travel-query.js` ✅ |

### 合法参数组合

| 组合 | 参数示例 | 说明 |
|------|---------|------|
| 目的地 | `--destination "三亚"` | 按城市/地区查询 |
| 目的地 + 假期 | `--destination "三亚" --extra "五一假期"` | 指定假期 |
| 目的地 + 天数 + 类型 | `--destination "云南" --extra "6 天 5 晚 自由行"` | 指定行程 |

### 参数说明

| 参数 | 说明 |
|------|------|
| `--destination <城市/地区>` | 目的地城市或地区(必填) |
| `--extra <补充信息>` | 额外信息(假期、天数、类型等) |

### extra 参数示例

- `"五一假期"` - 五一期间
- `"6 天 5 晚"` - 行程天数
- `"自由行"` - 自由行产品
- `"跟团游"` - 跟团游产品
- `"五一假期 6 天 5 晚 自由行"` - 多条件组合

### 调用示例

```bash

node scripts/travel-query.js --destination "三亚" --channel webchat --surface webchat

# 五一假期去三亚
node scripts/travel-query.js --destination "三亚" --extra "五一假期" --channel webchat --surface webchat

# 云南 6 天 5 晚自由行
node scripts/travel-query.js --destination "云南" --extra "6 天 5 晚 自由行" --channel webchat --surface webchat
```

- **禁止伪造不存在的链接**
