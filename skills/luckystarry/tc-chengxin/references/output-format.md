# 输出格式参考

_同程旅行数据输出格式示例_

---

## 📊 格式选择规则

| 条件 | 使用格式 | 链接形式 |
|------|---------|---------|
| `channel` = `webchat` | **表格格式** | Markdown 链接 `[PC 端](url)` |
| `surface` = `mobile` | **卡片格式** | Markdown 链接 `[手机端](url)` |
| `channel` 包含 `wechat`/`weixin`/`微信` | **卡片格式** | **纯文本链接**（直接显示 URL） |
| 其他所有情况 | **卡片格式** | Markdown 链接 `[PC 端](url)` |

**检测优先级：** 微信渠道 > webchat > surface=mobile > 默认

> ⚠️ **webchat 渠道优先使用表格格式**，所有资源类型（火车、机票、酒店、景区、长途汽车）都应支持表格输出。

## ⚠️ 预订链接强制要求

> 🚨 **链接是用户完成预订的唯一有效路径，必须完整输出！**

**要求：**
1. **每个资源都必须有预订链接**（车次、航班、酒店、景区、汽车班次等）
2. **同时输出 PC 端和手机端链接**（除非 API 只返回一种）
3. **禁止省略、替换或修改链接**
4. **禁止伪造不存在的链接**
5. 链接格式：
   - 普通渠道：`[PC 端](url)` | `[手机端](url)`
   - 微信渠道：`PC 端：url` | `手机端：url`

**错误示例** ❌：
```markdown
| 车次 | ... | 预订链接 |
| G123 | ... | [预订](#) |  <!-- 链接被替换 -->
```

**正确示例** ✅：
```markdown
| 车次 | ... | PC 预订 | 手机预订 |
| G123 | ... | [PC 端](https://www.ly.com/...) | [手机端](https://m.ly.com/...) |
```
